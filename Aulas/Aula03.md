
## Aula 03
### Revisão da Última Aula
- Software = Produto (materialidade; fabricação e desgaste)
  - conjunto de instruções
  - dados
  - documentação
- Engenharia = Organização para produzir algo com **qualidade**
  - comunicação: requisitos. *características
  - planejamento: prazo, custo e protótipo.
  - projeto: linguagem UML.
  - construção: código.
  - entrega: manutenção e documentação.
    - **Qualidade**
      - revisão e teste.
    - **Gestão**
      - acompanha o processo todo.
### Fluxo de Processo
- Fluxo Linear: seguir em sequencia as etapas, sem retorno as etapas anteriores; modelo *irreal*
- Fluxo Iterativo: é possível voltar de uma etapa para outra durante o seguimento de etapas; modelo *real*
### Fluxo incremental
- Ciclo de construção de algo = o projeto é o mesmo em diferentes fases
- Cada vez que uma parte é feita, outra é incrementada, conforme disponibilidade
- "Giramos" as etapas da Engenharia
- O primeiro incremento **precisa** ser utilizável
- Não funciona para todo produto
### Fluxo Evolucionário
- executa as atividades de uma
forma “circular”
- Cada volta pelas cinco atividades conduz a uma versão
mais completa do software.
### Fluxo Paralelo
- Efetuação de etapas sem antes ter passado pelas anteriores
### Metodologia / Ciclo de Vida de Desenvolvimento de Software
- Forma como a equipe resposável pelo projeto é chamada.
### Artefato e engenharia de software = uma forma de documento de requisitos
- Documento que contém letra e símbolos são chamados de artefato
- Código de software é um artefato
- Executável é um artefato
### Modelos de Processos Prescritivos - Tradicionais 
- **Cascata**
  - abordagem sequencial, linear e sistemátia
- **V**
  - ainda é linear
  - garantia da qualidade + etapas de desenvolvimento
  - acrescenta uma *camada de teste* entre cada etapa de desenvolvimento
- obs**: pesquisar camadas de teste
- **Processo Incremental**
  - fluxox lineares + paralelos
  - sequencias lineares de forma escalonada
  - sistma dividido em modulos
- **Processo evolucionário**
  - iterativos
  - **Modelos comuns nesse processo**:
    - Prototipação
      - iteração de prototipação é planejada rapidamente e ocorre a modelagem na forma de um “projeto rápido”
      - "casa com puxadinho", "remendado"
      - clientes podem achar que é uma versão operacional do software
    - Modelo Espiral
      - Acopla a natureza iterativa da prototipação com os aspectos sistemáticos e controlados do modelo cascata
      - Para cada módulo feito há uma análise de risco
- **Modelo Concorrente**
   - Todas as atividades de engenharia de software existem concorrentemente, porém estão em diferentes estados.
